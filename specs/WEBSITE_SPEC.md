# LinkedIn Challenge for Students - 웹사이트 스펙 문서

**최종 업데이트**: 2026-03-12
**목적**: 외국계 취업 링크드인 5주 챌린지 제안서 겸 홍보 페이지
**상태**: 인터뷰 완료, 개발 준비 단계

---

## 1. 프로젝트 개요

### 1.1 목표
- 학생 대상 링크드인 챌린지 프로그램 소개
- 프로그램 신뢰도 구축 및 참여 유도
- 제안서 겸 홍보 페이지로서의 역할

### 1.2 형태
- **정적 웹페이지** (초기 론칭)
- 향후 신청 기능, 가격 정보 등 추가 가능성 고려

### 1.3 예상 트래픽
- 월 1,000명 이하 (소규모)
- 동시 접속 사용자 최소화

---

## 2. 기술 스택 & 인프라

### 2.1 기술
- **프론트엔드**: HTML5 + CSS3 + Vanilla JavaScript
- **호스팅**: 정적 사이트 호스팅 (Vercel, Netlify, GitHub Pages 등)
- **콘텐츠**: Markdown + JavaScript로 파싱하여 렌더링

### 2.2 선택 이유
- 빠른 개발 & 배포
- 유지보수 간편
- 낮은 서버 비용
- 향후 기능 추가 용이

### 2.3 라이브러리 (검토 단계)
- **마크다운 파싱**: Marked.js 또는 Showdown.js
- **UI 컴포넌트**: 커스텀 (탭, 아코디언)
- **아이콘**: Font Awesome 또는 간단한 SVG

---

## 3. 정보 구조 & 콘텐츠 계층화

### 3.1 페이지 구조 (Top-to-Bottom)

| 순서 | 섹션 | 목적 | 우선순위 |
|------|------|------|---------|
| 1 | **Hero Section** | 프로그램 한눈에 파악 | 높음 |
| 2 | **커리큘럼** | 구체적인 학습 내용 이해 | **최우선** |
| 3 | **성과 사례** | 신뢰도 구축 (Before/After) | **최우선** |
| 4 | **프로그램 설명** | 상세한 프로그램 정보 | **높음** |
| 5 | **운영자 소개** | 신뢰성 강화 | 중간 |
| 6 | **FAQ** | 우려사항 해소 | 중간 |
| 7 | **CTA 영역** | 신청 유도 (향후 추가) | 낮음 |
| 8 | **Footer** | 연락처 & 추가 정보 | 낮음 |

### 3.2 사용자 여정 (User Journey)
```
방문 → 프로그램 이해 (커리큘럼) → 신뢰도 확인 (성과 사례) → 세부 정보 (설명) → 신청 결정
```

### 3.3 콘텐츠 출처
- **노션 글** (참고 자료): https://www.notion.so/... (제공된 이미지 기반)
- **마크다운 파일**: 각 섹션별 `.md` 파일로 분리 관리

## 4. 주요 섹션 상세 스펙

### 4.1 Hero Section (상단)
```
├─ 제목: "링크드인 5주 완성 스플린트"
├─ 부제: "외국계 취업을 위한 링크드인 전략"
├─ 핵심 메시지: "4주 후, 바뀌게 될 계정의 모습"
├─ 주요 수치 (3가지):
│  ├─ 팔로워 50-100명 증가
│  ├─ 게시글 평균 조회수 50회 이상
│  └─ 개인 맞춤형 링크드인 포스팅 완성
└─ 시각: 명확한 핵심 메시지 + 백그라운드 이미지 (선택)
```

### 4.2 커리큘럼 섹션 (최우선)
**목적**: 방문자가 "이 프로그램이 뭐 하는 건지" 명확히 이해

**구조**:
- 마크다운 형식으로 5주차 정보 표시
- **탭 또는 아코디언 토글** 방식
  - 탭: 1주차, 2주차, 3주차, 4주차, 5주차
  - 각 탭 내부:
    - 강의 주제
    - 미션 (bullet points)
    - 핵심 활동
    - 🎁 특별 혜택 (있는 경우)

**예시**:
```
📌 1주차: [Diagnosis] 이력서 진단 및 타겟 설정
├─ 강의: 채용 담당자의 눈을 사로잡는 링크드인 SEO...
├─ 미션:
│  ├─ 프로필 사진, 헤드라인, About 섹션 기초 세팅
│  └─ 국문/영문 이력서 제출
└─ 🎁 리드제일의 Kick: 이력서 분석 후 맞춤형 기업 3곳 + HR 담당자 리스트 제공
```

### 4.3 성과 사례 섹션 (최우선)
**목적**: 신뢰도 구축 (실제 결과 증명)

**구조**:
- **참여자 사례 3-4개** (노션 글 참고)
- 각 사례마다:
  - 참가자 이름 / 기간
  - Before/After 수치 비교
    - ✓ 팔로워 증가율
    - ✓ 게시글 조회수 증가율
    - ✓ 기타 성과
  - 짧은 후기 (1-2문장)

**예시**:
```
📊 참가자 사례 1
├─ 기간: 4주만에
├─ 팔로워: 32% 증가
├─ 게시글 조회수: 155.1% 증가
└─ 후기: "네 글을 투고하게 되었습니다"
```

**향후 추가 가능**:
- 동적 업데이트 기능 (Google Sheet 연동 또는 CMS)
- 현재: 고정 콘텐츠로 시작

### 4.4 프로그램 설명 섹션 (높음)
**목적**: 참여 조건, 방식, 혜택 상세 설명

**구조**:
- **참여 방식** (강의, 미션, 피드백 방식)
- **참가비 & 혜택** (향후 추가, 지금은 생략)
- **이런 분만 지원하세요** (적격 조건)
- **환불 정책**

**형식**: 마크다운 + 테이블 또는 리스트

### 4.5 운영자 소개 섹션 (중간)
**목적**: 운영자 신뢰성 강화

**구조**:
- 운영자 이름 & 링크드인 링크
- 주요 성과 (팔로워 수, 미디어 선정 등)
- 짧은 자기소개
- 프로필 사진 (선택)

### 4.6 FAQ 섹션 (중간 - 서브섹션)
**목적**: 일반적인 우려사항 해소

**구조**:
- 아코디언 토글 형식
- 주요 FAQ (노션 글 참고):
  - 초보자도 참여 가능?
  - 직장인도 참여 가능?
  - 녹화본 제공?
  - 질문은 언제?
  - 피드백은 어떻게?

### 4.7 Footer
```
├─ 연락처
│  ├─ 링크드인 DM: [링크]
│  └─ 이메일: leadjaeil@gmail.com
├─ 소셜 링크
│  └─ 링크드인, 인스타그램 등
└─ 저작권 표시
```

## 5. 디자인 & 브랜드 가이드

### 5.1 색상 팔레트
**기준**: 링크드인 색상 체계 적용

| 용도 | 색상 | Hex |
|------|------|-----|
| Primary (CTA, 헤더) | LinkedIn Blue | `#0A66C2` |
| Secondary | 밝은 회색 배경 | `#F3F5F7` |
| Accent | 초록색 (강조) | `#31A24C` |
| Text (본문) | 검정/진회색 | `#000000` / `#414141` |
| Text (보조) | 밝은 회색 | `#8A8D91` |

### 5.2 타이포그래피
- **헤더 폰트**: 현대적이고 명확한 산세리프 (예: Inter, Spoqa Han Sans)
- **본문 폰트**: 가독성 높은 산세리프 (예: Pretendard, -apple-system)
- **폰트 크기**:
  - H1 (Main Title): 48px
  - H2 (Section Title): 32px
  - Body: 16px
  - Small: 14px

### 5.3 레이아웃 & 여백
- **컨테이너 너비**: 1200px (데스크톱)
- **최소 여백 (padding)**: 40px (상하), 20px (좌우)
- **섹션 간 여백**: 60-80px

### 5.4 반응형 디자인
- **Primary**: Desktop (1200px 이상)
- **Secondary**: Tablet (768px - 1199px) - 간단한 최적화
- **Mobile**: 고려하지 않음 (향후 추가 가능)

### 5.5 디자인 톤
- **느낌**: 세련되고 깔끔한 (노션 스타일 참고)
- **감정**: 신뢰감, 프로페셔널함
- **이모지 사용**: 절제적 (섹션 제목, 리스트 항목에만)

### 6.2 스크롤 이벤트 (선택)

- Smooth scroll (anchor link)

### 6.3 하이퍼링크
- 외부 링크 (LinkedIn, 이메일): `target="_blank"` + 명확한 시각 표시
- 내부 링크 (섹션 이동): smooth scroll

## 7. 콘텐츠 관리 & 업데이트 전략

### 7.1 현재 (V1)
- 모든 콘텐츠: 마크다운 `.md` 파일
- 위치: `src/content/*.md`
- 업데이트 방식: 코드 수정 + 배포

### 7.2 향후 (V2+)
**고려 옵션**:
1. **Google Sheets 연동** (참가자 후기, 성과 사례)
   - 비개발자도 수정 가능
   - 초기 개발 복잡도 증가

2. **간단한 관리자 페이지** (비밀번호 로그인)
   - Firebase 등으로 데이터 저장
   - CRUD 페이지 필요

3. **헤드리스 CMS** (Contentful, Sanity)
   - 프로페셔널하지만 초과 스펙

**현재 권고**: V1은 정적 콘텐츠로 시작 → V2에서 업데이트 시스템 추가
